# git tips that save time

not "use git." not "commit often." things i do daily that save real time.

## `git commit --amend --no-edit`

wrote a commit, immediately realized you forgot a file? don't make a new commit. amend the last one.

```bash
git add forgotten-file.js
git commit --amend --no-edit
```

keeps the history clean. one logical change, one commit.

## `git stash -u`

regular `git stash` only stashes tracked files. the `-u` flag also stashes untracked files. the thing you created and forgot to `git add`? saved.

## `git log --oneline --graph --all -20`

visualizes the last 20 commits across all branches. great for understanding what happened while you weren't looking.

## `git diff --stat`

shows you *what* changed without the actual diff. "3 files changed, 47 insertions, 12 deletions." perfect for a quick sanity check before committing.

## `git bisect`

binary search through your commit history to find which commit introduced a bug.

```bash
git bisect start
git bisect bad          # current commit is broken
git bisect good abc123  # this commit was fine
# git will checkout a commit. test it.
# tell it good or bad.
# repeat until it finds the culprit.
```

sounds niche until you need it. when you need it, it saves hours.

## `git worktree`

need to work on two branches at the same time? `git worktree` lets you check out a branch into a separate directory without switching.

```bash
git worktree add ../my-repo-fixes fix/login-bug
cd ../my-repo-fixes
# now you're on fix/login-bug without leaving your current branch
```

huge when you're mid-task and get asked to review a PR or fix something urgent.

## `git rebase -i HEAD~5`

interactive rebase for the last 5 commits. squash, reorder, edit, or drop commits. this is how you turn "fixed typo," "fixed another typo," "actually fixed it for real" into one clean commit.

use on branches you haven't pushed. if you've pushed and someone else pulled, use `merge` instead.

## `git reflog`

think you lost a commit? `git reflog` shows you every HEAD movement, including the ones you "deleted." you can almost always recover from anything with reflog.

```bash
git reflog
# find the commit hash you want
git checkout <hash>
```

## `git push --force-with-lease`

like `--force`, but safer. it won't overwrite commits you haven't seen. if someone pushed while you weren't looking, it refuses instead of destroying their work.

use this instead of `--force`. always.

## one more: `git log -p --since="yesterday" --author=you`

shows all your commits from yesterday with full diffs. great for writing end-of-day summaries or figuring out "what did i even do today?"
